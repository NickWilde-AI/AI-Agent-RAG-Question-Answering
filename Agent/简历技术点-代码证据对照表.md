# 简历技术点-代码证据对照表（一页纸）

> 用法：面试时每讲一个技术点，顺手报出“文件路径”，可信度会明显提升。

## 1) Agent 闭环 / ReAct 思路
- 技术点：Plan-Execute-Verify-Retry 闭环
- 代码证据：`src/agent_loop.py`、`src/api.py`（返回 `loop_steps`）

## 2) 检索工程优化
- 技术点：query rewrite、类型预过滤、混合打分
- 代码证据：`src/retriever.py`

## 3) Router Agent（多工具分发）
- 技术点：LLM 路由 + 规则兜底，四大分支路由
- 代码证据：`src/router.py`

## 4) 多工具编排
- 技术点：`fact_qa` / `multi_page_qa` / `chart_qa` / `translate_qa`
- 代码证据：`src/tools.py`、`src/pipeline.py`

## 5) Verifier 可证性校验 + fallback
- 技术点：答案证据校验，失败扩 top-k 重试
- 代码证据：`src/verifier.py`、`src/pipeline.py`

## 6) 会话缓存（Session Memory）
- 技术点：历史 QA 与命中页缓存
- 代码证据：`src/memory.py`
- Redis 版本证据：`src/infra/redis_memory.py`

## 7) Milvus 向量库能力
- 技术点：向量库抽象 + Milvus 适配
- 代码证据：`src/infra/vector_store.py`

## 8) PDF 建库入口（PyMuPDF）
- 技术点：PDF 按页读取并产出页面数据结构
- 代码证据：`src/infra/pdf_ingest.py`

## 9) FastAPI 服务化与监控
- 技术点：`/ask`、`/health`、`/metrics`
- 代码证据：`src/api.py`

## 10) 离线评测体系
- 技术点：Recall@10、Accuracy 指标实现与输出
- 代码证据：`src/eval_metrics.py`、`main.py`

## 11) 配置化治理（灰度/切换）
- 技术点：开关化配置（LLM、Loop、Milvus、Redis）
- 代码证据：`src/config.py`、`.env`
